# landing-lab2

Односторінковий лендінг для представлення бакалаврської роботи, виконаний у межах лабораторної роботи №2 з використанням **GitHub Flow** як стратегії гілкування.

## Про проєкт

Цей проєкт є лендінг-сторінкою для бакалаврської роботи на тему:

**Інформаційна вебсистема для створення, модерації та оцінювання авторських літературних творів**  
**Information Web System for Creating, Moderating, and Evaluating Original Literary Works** :contentReference[oaicite:0]{index=0}

На сторінці представлено короткий опис роботи, актуальність теми, ключові слова, мету дослідження, основні завдання, методологію, очікувані результати, демонстраційні матеріали та контактну інформацію автора. :contentReference[oaicite:1]{index=1}

## Мета лабораторної роботи

Розробити сучасний адаптивний лендінг для бакалаврської роботи з дотриманням вимог до:

- семантичної HTML-структури
- SEO-оптимізації
- доступності
- адаптивності
- організації історії змін у Git

## Обрана стратегія гілкування

### GitHub Flow

У цій роботі використано стратегію **GitHub Flow**.

Коротко про підхід:
- основна стабільна гілка, `main`
- для кожної нової задачі створюється окрема feature-гілка
- зміни вносяться невеликими логічними комітами
- після завершення роботи зміни зливаються в `main` через pull request

Цей підхід добре підходить для невеликих вебпроєктів, дозволяє зручно контролювати історію змін і підтримувати стабільний стан основної гілки.

## Основний функціонал сторінки

На лендінгу реалізовано:

- назву роботи українською та англійською мовами
- короткий опис бакалаврської роботи
- секцію з актуальністю теми
- ключові слова
- мету дослідження
- основні завдання
- опис методології дослідження
- очікувані результати
- демонстраційні зображення прототипу
- контактну інформацію автора
- посилання на репозиторій

## SEO та доступність

У проєкті реалізовано базові вимоги до SEO та accessibility:

- `meta description`
- `meta keywords`
- `meta author`
- Open Graph теги
- `robots.txt`
- `sitemap.xml`
- favicon для різних пристроїв
- семантичні теги `header`, `main`, `section`, `footer`
- `aria-label` атрибути
- `alt` для зображень
- Schema.org JSON-LD розмітка :contentReference[oaicite:2]{index=2}
